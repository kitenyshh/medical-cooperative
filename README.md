# Медицинский кооператив — информационная система

Курсовой проект по дисциплине «Создание программного обеспечения».  
Тема: разработка информационной системы медицинского кооператива с базой данных пациентов, осмотров, диагнозов, лекарств и отчётностью.

## Стек технологий

- Backend: Node.js, Express
- База данных: SQLite
- Шаблоны представления: EJS
- Frontend: HTML, CSS, нативный JavaScript

## Как запустить проект

git clone https://github.com/kitenyshh/medical-cooperative.git
cd medical-cooperative
npm install
node server.js

После запуска сервер будет доступен по адресу:  
`http://localhost:3000`

Начальный тестовый пользователь:

- логин: `doctor1`
- пароль: `doctor123`

Можно также зарегистрировать новых врачей через интерфейс.

## Структура базы данных

Основные таблицы:

- `doctors` — врачи (логин, пароль, ФИО).
- `patients` — пациенты (ФИО, пол, дата рождения, домашний адрес).
- `diagnoses` — справочник диагнозов.
- `visits` — осмотры пациентов врачами (дата, место, симптомы, диагноз, предписания).
- `medicines` — лекарства (название, способ приёма, действие, побочные эффекты).
- `prescriptions` — назначенные лекарства в рамках осмотров.

Дополнительно созданы представления (пакет функций):

- `v_visits_by_date` — количество вызовов по датам.
- `v_patients_by_diagnosis` — количество пациентов по диагнозам.
- `v_medicine_effects` — побочные эффекты по каждому лекарству.

## Реализованный функционал

### 1. Управление врачами

- Регистрация врача (логин, пароль, ФИО).
- Авторизация и выход из системы.

### 2. Управление пациентами

- Добавление новых пациентов с указанием ФИО, пола, даты рождения и домашнего адреса.
- Просмотр списка пациентов.
- Карточка пациента с историей приёмов, диагнозов и назначенных лекарств.
- Добавление нового заболевания (нового осмотра) из карточки пациента.

### 3. Проведение осмотра

Для каждого осмотра врач указывает:

- пациента;
- дату и место (приём / на дому);
- симптомы;
- диагноз (из справочника);
- текст предписаний;
- назначенные лекарства:
  - выбор из уже существующих в БД;
  - либо добавление нового лекарства (название, способ приёма, действие, побочные эффекты) с привязкой к осмотру.

### 4. Справочник диагнозов

- Просмотр списка диагнозов.
- Добавление нового диагноза, который затем доступен в формах осмотра и карточке пациента.

### 5. Отчётность

Отдельная страница отчётов реализует пункты задания:

1. **Вызовы по дате** — по выбранной дате отображается количество осмотров и список врачей, проводивших приёмы.
2. **Больные по диагнозу** — количество уникальных пациентов, заболевших каждой болезнью.
3. **Лекарства и побочные эффекты** — таблица лекарств с указанием способа приёма, действия и возможных побочных эффектов.

Фильтрация по дате и другим параметрам выполняется с использованием SQL‑запросов и представлений.

## Назначение проекта

Система демонстрирует:

- анализ предметной области медицинского кооператива;
- проектирование и реализацию реляционной базы данных;
- использование пакета SQL‑функций/представлений для отчётности;
- разработку веб‑интерфейса для работы коллектива врачей с данными пациентов и лекарственных назначений.
